Hayasaka Reiichiro Engineering Notes

🌸 SpringBoot 是什麼？

SpringBoot 本質上：
不是單純「Java Web 框架」。

而是：
快速建立後端伺服器的工業化工具箱。 

React / Vue
      ↓
SpringBoot
      ↓
MySQL

SpringBoot 負責：
✔ 接收請求
✔ 處理資料
✔ 存取資料庫
✔ 回傳結果

📦 專案結構

src/
 └── main/
      ├── java/
      │    └── com/example/
      │         ├── controller/
      │         ├── service/
      │         ├── mapper/
      │         └── entity/
      │
      └── resources/
            ├── application.yml
            └── mapper/

🚪 Controller（控制器）

Controller 可以理解成：
網站大門。 

@GetMapping("/user")

意思：
有人訪問 /user
↓
Controller 接收請求。

🧠 Service（業務邏輯層）

Service 是：
真正處理邏輯的地方。

例如：
✔ 登入
✔ 註冊
✔ 權限驗證
✔ 訂單計算

Controller 不應該塞大量邏輯。
因為：
Controller 是門口，Service 才是辦公室。

🛰️ Mapper（資料庫對映）

Mapper 本質：
Java 與 MySQL 的翻譯官。 

userMapper.selectById(1)

意思：
去資料庫查詢 id=1 的資料。

📘 Entity（實體類）

Entity 是：
資料模型。 

public class User

代表資料庫中的 user 表。 

id | name | age
----------------
1  | Rei  | 20

⚙ resources（設定區）

最重要的檔案：
application.yml 

server:
  port: 8080

可以設定：
✔ 伺服器 Port
✔ 資料庫
✔ SpringBoot 設定

📜 pom.xml 是什麼？

pom.xml 的核心概念：
告訴 Maven 需要安裝哪些功能。 

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

意思：
安裝 SpringBoot Web 功能。

🌌 SpringBoot 執行流程（超重要）

瀏覽器
   ↓
Controller
   ↓
Service
   ↓
Mapper
   ↓
MySQL
   ↓
回傳資料

這條流程就是：
整個後端系統的主幹。

⚡ 為什麼以前 PHP 網站容易卡？

很多早期網站：
PHP + MySQL

沒有：
❌ 分頁
❌ 快取
❌ 索引
❌ 分層

所以文章一多：
網站就爆炸。

現在大型網站通常會使用：
✔ Redis 快取
✔ Docker 部署
✔ JWT 驗證
✔ MyBatis 對映
✔ Nginx 反向代理
