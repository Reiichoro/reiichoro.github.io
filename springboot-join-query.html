② SQL建表結構
🗄️ CREATE TABLE
-- 用戶表 CREATE TABLE users ( id BIGINT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50) NOT NULL, email VARCHAR(100) UNIQUE, phone VARCHAR(20), created_at DATETIME DEFAULT CURRENT_TIMESTAMP ); -- 訂單表（user_id 關聯 users.id） CREATE TABLE orders ( id BIGINT PRIMARY KEY AUTO_INCREMENT, user_id BIGINT NOT NULL, ← 外鍵：關聯用戶 order_no VARCHAR(30) UNIQUE, total_amount DECIMAL(10,2), status VARCHAR(20) DEFAULT 'pending', created_at DATETIME DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ); -- 訂單項表（order_id 關聯 orders.id） CREATE TABLE order_items ( id BIGINT PRIMARY KEY AUTO_INCREMENT, order_id BIGINT NOT NULL, ← 外鍵：關聯訂單 product_name VARCHAR(100), quantity INT DEFAULT 1, price DECIMAL(10,2), FOREIGN KEY (order_id) REFERENCES orders(id) );
③ JOIN類型選擇
INNER JOIN（內聯）

只返回兩表都有匹配記錄的行。
沒有訂單的用戶不會出現。

用戶 ∩ 訂單（交集）
LEFT JOIN（左聯）

返回左表所有行，右表無匹配則NULL。
沒有訂單的用戶也會顯示。

所有用戶，訂單可為空
本例用LEFT JOIN

查詢滿足條件的訂單及用戶信息——用LEFT JOIN確保即使訂單沒有訂單項也能查出來。

orders LEFT JOIN items
④ 核心SQL — 三表聯查
🔍 場景1：查詢滿足條件的訂單及用戶信息、訂單項
-- 查詢特定用戶的所有訂單及訂單項（三表聯查） SELECT -- 用戶信息 u.id AS user_id, u.name AS user_name, u.email AS user_email, -- 訂單信息 o.id AS order_id, o.order_no AS order_no, o.total_amount AS total_amount, o.status AS order_status, o.created_at AS order_date, -- 訂單項信息 oi.id AS item_id, oi.product_name AS product_name, oi.quantity AS quantity, oi.price AS price FROM orders o INNER JOIN users u ON o.user_id = u.id ← 用戶ID關聯 LEFT JOIN order_items oi ON oi.order_id = o.id ← 訂單ID關聯 WHERE o.status = 'completed' ← 只查已完成訂單 AND o.total_amount > 100 ← 金額滿100 AND u.id = #{userId} ← 指定用戶（參數） ORDER BY o.created_at DESC;
🔍 場景2：查詢所有用戶及其訂單統計（聚合）
-- 查詢每個用戶的訂單數和總消費金額 SELECT u.id, u.name, COUNT(o.id) AS order_count, ← 訂單數量 SUM(o.total_amount) AS total_spent, ← 總消費 MAX(o.created_at) AS last_order_date ← 最後一次下單 FROM users u LEFT JOIN orders o ON o.user_id = u.id GROUP BY u.id, u.name HAVING COUNT(o.id) > 0 ← 只要有訂單的用戶 ORDER BY total_spent DESC;

⑥ MyBatis ResultMap — 最核心的映射配置
💡 為什麼需要ResultMap？
SQL聯查返回的是「扁平的行數據」（每行都有user+order+item的欄位），但Java需要的是「嵌套的物件結構」（User裡面有List<Order>，Order裡面有List<OrderItem>）。
ResultMap就是告訴MyBatis：如何把扁平行數據組裝成嵌套物件。
📄 UserMapper.xml — ResultMap定義
<resultMap id="UserOrderResultMap" type="UserOrderVO"> <!-- 用戶基本信息映射 --> <id property="userId" column="user_id"/> <result property="userName" column="user_name"/> <result property="userEmail" column="user_email"/> <!-- 聯結的訂單列表（一對多） --> <collection property="orders" ← 對應Java屬性名 ofType="OrderDetailVO" ← 列表元素的類型 resultMap="OrderDetailResultMap"/> </resultMap> <!-- 訂單詳情ResultMap（嵌套） --> <resultMap id="OrderDetailResultMap" type="OrderDetailVO"> <id property="orderId" column="order_id"/> <result property="orderNo" column="order_no"/> <result property="totalAmount" column="total_amount"/> <result property="status" column="order_status"/> <result property="orderDate" column="order_date"/> <!-- 聯結的訂單項列表（一對多） --> <collection property="items" ofType="OrderItemVO"> <id property="itemId" column="item_id"/> <result property="productName" column="product_name"/> <result property="quantity" column="quantity"/> <result property="price" column="price"/> </collection> </resultMap> <!-- 使用ResultMap的查詢 --> <select id="getUserOrders" resultMap="UserOrderResultMap"> SELECT u.id AS user_id, u.name AS user_name, u.email AS user_email, o.id AS order_id, o.order_no, o.total_amount, o.status AS order_status, o.created_at AS order_date, oi.id AS item_id, oi.product_name, oi.quantity, oi.price FROM orders o INNER JOIN users u ON o.user_id = u.id LEFT JOIN order_items oi ON oi.order_id = o.id WHERE o.status = #{status} AND o.total_amount > #{minAmount} ORDER BY o.created_at DESC </select>

⚠️ MyBatis ResultMap的關鍵規則：
1. <id> 必須配置——MyBatis靠id欄位判斷「這兩行是否屬於同一個用戶/訂單」，用來合並重複行
2. SQL返回的是扁平數據（多行），ResultMap負責把相同user_id的行合並成一個User物件
3. <collection> 對應一對多，<association> 對應一對一
