① 認證（Authentication）vs 授權（Authorization）

🔑 認證（Authentication）

❓你是誰？ 驗證身份
📋用戶名＋密碼登錄
🎫驗證成功 → 發放Token（JWT）
🏨類比：酒店前台核對身份，發房卡
🔄每次請求攜帶Token，系統識別身份
VS

🛡️ 授權（Authorization）

❓你能做什麼？ 驗證權限
👤根據角色（ROLE_ADMIN/USER）決定
🚪有權限 → 放行 ／ 無權限 → 403
🏨類比：房卡只能開自己的房間
⚡認證之後才能做授權檢查
💡 順序很重要：先認證，再授權。
沒有登錄（認證失敗）→ 返回 401 Unauthorized（未認證）
登錄了但沒有權限（授權失敗）→ 返回 403 Forbidden（已認證但無權）
② 權限控制流程 — HTTP請求通過安全過濾器鏈
🌐 HTTP Request
客戶端發送請求，Header中帶有 Authorization: Bearer {token}
↓ 進入Spring Security過濾器鏈
CorsFilter
處理跨域請求（CORS），允許前端域名訪問
↓
CsrfFilter
防止跨站請求偽造攻擊。REST API通常關閉此過濾器
↓
⭐ JwtAuthFilter（自定義）
最關鍵！從Header取出JWT Token → 驗證Token有效性 → 解析用戶信息 → 放入SecurityContext
↓ Token有效，認證信息已放入上下文
UsernamePasswordAuthenticationFilter
處理表單登錄（用戶名密碼）。已用JWT時一般跳過
↓
⭐ AuthorizationFilter（權限檢查）
關鍵！讀取SecurityContext中的用戶角色 → 對比請求路徑所需權限 → 有權限放行，無權限返回403
↓ 通過所有過濾器
✅ Controller（目標方法）
請求成功抵達Controller，執行業務邏輯，返回響應
兩條拒絕路徑：
JWT驗證失敗（Token過期/無效）→ 直接返回 401，不繼續執行
JWT有效但角色不夠 → AuthorizationFilter返回 403
③ JWT Token — 認證信息的載體

🔐 登錄流程（獲取Token）

1.前端發送 POST /login {username, password}
2.Security驗證用戶名密碼是否正確
3.正確 → 生成JWT Token（包含userId、roles、過期時間）
4.返回Token給前端
5.前端保存Token（localStorage或Cookie）

🔓 後續請求（使用Token）

1.前端每次請求帶上Header：
Authorization: Bearer eyJhbGci...
2.JwtAuthFilter解析Token，取出用戶信息
3.放入SecurityContext，後續可直接獲取當前用戶
4.Token過期（通常7天）→ 重新登錄
🔑 JWT Token結構（三段Base64）
// JWT = Header.Payload.Signature（用.分隔） eyJhbGciOiJIUzI1NiJ9 ← Header（算法） . eyJ1c2VySWQiOjEsInJvbGVzIjpbIlJPTEVfQURNSU4iXSwiZXhwIjoxNzE2MDAwMDAwfQ== ← Payload（解碼後）：{"userId":1, "roles":["ROLE_ADMIN"], "exp":1716000000} . HMACSHA256(base64(header)+"."+base64(payload), secretKey) ← Signature（防篡改）
④ 角色與權限設計
角色 查看用戶列表 創建用戶 刪除用戶 查看訂單 管理系統配置
ROLE_ADMIN ✅ ✅ ✅ ✅ ✅
ROLE_MANAGER ✅ ✅ ❌ ✅ ❌
ROLE_USER ❌ ❌ ❌ ✅（只看自己） ❌
⑤ 完整代碼實現
⚙️ SecurityConfig — 安全配置
@Configuration @EnableMethodSecurity ← 開啟方法級別權限控制 public class SecurityConfig { @Autowired private JwtAuthFilter jwtAuthFilter; @Bean public SecurityFilterChain filterChain(HttpSecurity http) throws Exception { http // 關閉CSRF（REST API不需要） .csrf(csrf -> csrf.disable()) // 設置各路徑的訪問規則 .authorizeHttpRequests(auth -> auth // 這些路徑不需要登錄 .requestMatchers("/api/auth/**").permitAll() .requestMatchers("/api/public/**").permitAll() // 管理員才能訪問 .requestMatchers("/api/admin/**").hasRole("ADMIN") // 其他所有路徑都需要登錄 .anyRequest().authenticated() ) // 無狀態Session（JWT不需要Session） .sessionManagement(s -> s .sessionCreationPolicy(SessionCreationPolicy.STATELESS)) // 在UsernamePassword過濾器之前插入JWT過濾器 .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class); return http.build(); } }
🔐 JwtAuthFilter — 認證信息放入上下文
@Component @Slf4j public class JwtAuthFilter extends OncePerRequestFilter { @Autowired private JwtUtils jwtUtils; @Override protected void doFilterInternal(HttpServletRequest req, HttpServletResponse res, FilterChain chain) throws Exception { // 1. 從Header取Token String header = req.getHeader("Authorization"); // 2. 格式驗證 if (header == null || !header.startsWith("Bearer ")) { chain.doFilter(req, res); ← 無Token，繼續（後面會被攔截） return; } String token = header.substring(7); ← 去掉"Bearer "前綴 try { // 3. 驗證Token並解析用戶信息 Claims claims = jwtUtils.parseToken(token); Long userId = claims.get("userId", Long.class); List<String> roles = claims.get("roles", List.class); // 4. 構建認證信息（核心！） List<GrantedAuthority> authorities = roles.stream() .map(SimpleGrantedAuthority::new) .collect(Collectors.toList()); UsernamePasswordAuthenticationToken authentication = new UsernamePasswordAuthenticationToken(userId, null, authorities); // 5. 放入SecurityContext（關鍵！後續過濾器和Controller都能獲取） SecurityContextHolder.getContext().setAuthentication(authentication); log.debug("JWT驗證成功, userId={}, roles={}", userId, roles); } catch (ExpiredJwtException e) { log.warn("Token已過期"); res.sendError(401, "Token已過期，請重新登錄"); return; } catch (Exception e) { log.warn("Token無效"); res.sendError(401, "Token無效"); return; } chain.doFilter(req, res); ← 繼續執行後面的過濾器 } }
🛡️ Controller — 權限檢查注解
@RestController @RequestMapping("/api/users") public class UserController { // 所有登錄用戶可以訪問 @GetMapping("/{id}") public Result<UserVO> getUser(@PathVariable Long id) { return Result.success(userService.findById(id)); } // 只有ADMIN可以訪問 @GetMapping @PreAuthorize("hasRole('ADMIN')") ← 方法級別權限控制 public Result<PageResult<UserVO>> getAllUsers(...) { ... } // ADMIN或MANAGER可以訪問 @PostMapping @PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')") public Result<UserVO> createUser(@RequestBody UserDTO dto) { ... } // 只有ADMIN可以刪除 @DeleteMapping("/{id}") @PreAuthorize("hasRole('ADMIN')") public Result<Void> deleteUser(@PathVariable Long id) { ... } // 在方法內獲取當前登錄用戶 @GetMapping("/me") public Result<UserVO> getCurrentUser( @AuthenticationPrincipal Long userId ← 自動注入當前登錄的userId ) { return Result.success(userService.findById(userId)); } }
🔑 AuthController — 登錄接口
@RestController @RequestMapping("/api/auth") public class AuthController { @PostMapping("/login") public Result<String> login(@RequestBody LoginDTO dto) { // 1. 驗證用戶名密碼 User user = userService.findByUsername(dto.getUsername()); if (user == null || !passwordEncoder.matches(dto.getPassword(), user.getPassword())) { throw new BusinessException(ErrorCode.LOGIN_FAILED); } // 2. 生成JWT Token（包含userId和角色列表） String token = jwtUtils.generateToken(user.getId(), user.getRoles()); // 3. 返回Token return Result.success(token); } }
⑥ 完整請求流程總結
// 場景：ROLE_USER嘗試訪問 DELETE /api/users/1（需要ADMIN） 客戶端 → DELETE /api/users/1 Header: Authorization: Bearer {token} ↓ JwtAuthFilter： 解析Token → userId=5, roles=["ROLE_USER"] 放入SecurityContext ✅ ↓ AuthorizationFilter： 檢查路徑 DELETE /api/users/** → 需要 ROLE_ADMIN 當前用戶只有 ROLE_USER ❌ 權限不足 → 返回 403 Forbidden ↓ 前端收到： { "code": 403, "message": "無權限訪問" } // 場景：ROLE_ADMIN訪問同一接口 JwtAuthFilter：userId=1, roles=["ROLE_ADMIN"] ✅ AuthorizationFilter：ROLE_ADMIN 匹配 ✅ Controller.deleteUser()執行 ✅ 返回 200 success ✅
