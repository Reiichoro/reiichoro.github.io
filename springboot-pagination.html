Spring Boot 分頁查詢完全版
分頁流程 · 前端請求 · 後端實現 · 返回數據結構
① 什麼是分頁查詢？為什麼需要？
💡 問題背景：數據庫有100萬條用戶記錄，一次性全部查出來返回給前端 → 內存爆炸、網路超時、頁面卡死。

解決方案：分頁查詢——每次只查一頁的數據（比如每頁20條），前端翻頁時再請求下一頁。
就像書本的頁碼：一次只看一頁，需要時才翻頁。
pageNum
當前頁碼
從1開始
例：第3頁
pageSize
每頁條數
通常10或20
例：每頁20條
total
總記錄數
讓前端知道
共有多少頁
pages
總頁數
= ceil(total/pageSize)
例：50頁
② 分頁查詢完整流程

一次分頁請求的完整旅程

前端（Browser）
用戶點擊「第2頁」，發送請求：GET /api/users?pageNum=2&pageSize=20
↓
Controller層
接收請求參數：用@RequestParam接收pageNum和pageSize，傳給Service
↓
Service層
構建分頁對象：創建PageHelper.startPage(pageNum, pageSize)或PageRequest，調用Mapper查詢
↓
Mapper / Repository
執行SQL：MyBatis自動在SQL後加上LIMIT 20 OFFSET 20（第2頁從第21條開始）
↓
數據庫
執行：SELECT * FROM users LIMIT 20 OFFSET 20
同時執行：SELECT COUNT(*) FROM users（算總數）
↑（返回數據）
Service層
封裝分頁結果：把數據列表、總數、總頁數封裝成PageResult物件返回
↑
Controller層
返回JSON響應給前端，包含數據列表和分頁信息
↑（HTTP響應）
前端（Browser）
收到JSON，渲染表格數據，更新分頁導航（共50頁，當前第2頁）
③ 前端請求格式
🌐 前端發送的請求
// 基本分頁請求 GET /api/users?pageNum=1&pageSize=20 // 帶條件的分頁查詢（搜索+分頁） GET /api/users?pageNum=1&pageSize=20&name=Alice&status=active // 帶排序的分頁查詢 GET /api/users?pageNum=2&pageSize=10&sortBy=createdAt&sortDir=desc // POST請求（複雜查詢條件放Body） POST /api/users/page Content-Type: application/json { "pageNum": 1, "pageSize": 20, "name": "Alice", "startDate": "2024-01-01", "endDate": "2024-12-31" }
④ 返回的分頁數據結構（JSON）
// 標準分頁響應結構 { "code": 200, ← 狀態碼 "message": "success", ← 提示信息 "data": { "records": [ ← 當前頁的數據列表 { "id": 21, "name": "Alice", "email": "alice@example.com", "createdAt": "2024-03-15" }, // ... 共20條 ], "pageNum": 2, ← 當前頁碼 "pageSize": 20, ← 每頁條數 "total": 1000, ← 總記錄數 "pages": 50, ← 總頁數 = ceil(1000/20) "hasPrevious": true, ← 是否有上一頁 "hasNext": true ← 是否有下一頁 } }

🔑 各字段的用途說明

"records": [...]
當前頁的實際數據，前端渲染表格用這個
"total": 1000
總記錄數，告訴前端「共有多少條數據」
"pages": 50
總頁數，前端分頁導航需要這個數字
"pageNum": 2
當前第幾頁，前端高亮顯示當前頁用
"hasNext": true
是否有下一頁，控制「下一頁」按鈕是否可點
"hasPrevious": true
是否有上一頁，控制「上一頁」按鈕是否可點
⑤ 後端完整實現代碼
📦 PageRequest（請求DTO）
@Data public class PageRequest { private Integer pageNum = 1; ← 默認第1頁 private Integer pageSize = 20; ← 默認每頁20條 private String name; ← 查詢條件（可選） private String status; }
📤 PageResult（響應DTO）
@Data @AllArgsConstructor public class PageResult<T> { private List<T> records; ← 數據列表（泛型，可復用） private Long total; ← 總記錄數 private Integer pages; ← 總頁數 private Integer pageNum; ← 當前頁 private Integer pageSize; ← 每頁條數 private Boolean hasNext; ← 是否有下一頁 private Boolean hasPrevious; ← 是否有上一頁 }
🌐 Controller層
@RestController @RequestMapping("/api/users") public class UserController { @Autowired private UserService userService; // GET請求分頁（簡單查詢） @GetMapping public Result<PageResult<UserVO>> getUsers( @RequestParam(defaultValue = "1") Integer pageNum, @RequestParam(defaultValue = "20") Integer pageSize, @RequestParam(required = false) String name ) { return Result.success(userService.getUsers(pageNum, pageSize, name)); } // POST請求分頁（複雜查詢條件） @PostMapping("/page") public Result<PageResult<UserVO>> pageUsers( @RequestBody PageRequest request ) { return Result.success(userService.pageUsers(request)); } }
⚙️ Service層（使用MyBatis PageHelper）
@Service public class UserServiceImpl implements UserService { @Autowired private UserMapper userMapper; @Override public PageResult<UserVO> getUsers(Integer pageNum, Integer pageSize, String name) { // ⭐ 關鍵：在查詢前設置分頁，PageHelper會自動加LIMIT PageHelper.startPage(pageNum, pageSize); // 執行查詢（PageHelper自動攔截，加上LIMIT OFFSET） List<User> users = userMapper.findByName(name); // 轉換為PageInfo，包含分頁信息 PageInfo<User> pageInfo = new PageInfo<>(users); // 封裝返回結果 return new PageResult<>( convertToVO(pageInfo.getList()), ← 轉換為VO pageInfo.getTotal(), pageInfo.getPages(), pageInfo.getPageNum(), pageInfo.getPageSize(), pageInfo.isHasNextPage(), pageInfo.isHasPreviousPage() ); } }
🗃️ Mapper層（SQL）
// UserMapper.java @Mapper public interface UserMapper { // 普通查詢，PageHelper自動加LIMIT List<User> findByName(@Param("name") String name); } // UserMapper.xml（SQL定義） <select id="findByName" resultType="User"> SELECT * FROM users <where> <if test="name != null and name != ''"> AND name LIKE CONCAT('%', #{name}, '%') </if> </where> ORDER BY created_at DESC ← PageHelper自動在這裡加：LIMIT 20 OFFSET 20 </select>
⑥ 需要的依賴（pom.xml）
<!-- MyBatis分頁插件 PageHelper --> <dependency> <groupId>com.github.pagehelper</groupId> <artifactId>pagehelper-spring-boot-starter</artifactId> <version>1.4.6</version> </dependency> <!-- application.yml配置 --> pagehelper: helper-dialect: mysql ← 數據庫類型 reasonable: true ← 頁碼超出自動修正 support-methods-arguments: true
💡 PageHelper的魔法：
PageHelper.startPage(2, 20) 調用後，下一條執行的SQL會自動被攔截，加上 LIMIT 20 OFFSET 20。
同時自動執行 SELECT COUNT(*) 來計算總數。
開發者只需要寫普通查詢SQL，分頁邏輯完全由PageHelper處理！

// 請求分頁數據 async function loadUsers(pageNum = 1) { const response = await fetch( `/api/users?pageNum=${pageNum}&pageSize=20` ); const result = await response.json(); const { records, total, pages, hasNext, hasPrevious } = result.data; // 渲染表格數據 renderTable(records); // 渲染分頁導航 renderPagination({ currentPage: pageNum, totalPages: pages, ← 共50頁 total: total, ← 共1000條 hasNext: hasNext, ← 控制「下一頁」按鈕 hasPrevious: hasPrevious ← 控制「上一頁」按鈕 }); } // 點擊下一頁 nextPageBtn.onclick = () => loadUsers(currentPage + 1); prevPageBtn.onclick = () => loadUsers(currentPage - 1);
